Within my CSS code there was a stray curly bracket which I must have missed when removing a piece of code. I simply removed it and ran it back through the CSS Validator

Week 6 - Responsive Webpage
Published on Wednesday, November 23

This week I began making my webpage responsive.

  • Creating a webpage that is responsive to the user screen size is a concept I have not visited before. Ensuring my webpage is responsive while also following the correct standards was a difficult task. This is where I feel I faced the most challenges.
  • I began by reading through the the provided Lab Index to see how I should approach making my webpage responsive. The previous week I had succesfully formatted my webpage using the Grid Layout which was helpful as now all I had to do this week was incorporate Media Queries into my CSS.
    The use of Media Queries allows your webpage to become responsive when it detects a change on the users device. One example of this is changing the layout and presentation of your webpage depending on the size of the users viewport.
  • My choice of incorporating 2 columns within my grid layout allowed me to make a 'simple' change in my CSS in order for it to display appropriately on a smaller viewport (such as a mobile device). I implemented a media quiery that states if the width of the screen exceeds 768px (the greatest breakpoint of a tablet) the webpage will switch to its 'large screen' version and display the full two columns. However, if the pixels go below 768, the webpage switches to a version which displays everything within one column, using the block display.
  • I ran into some problems when it came to creating the mobile version.
  • The first problem I faced was scaling all of my fonts down equally across my website to the correct size so that they look how they were intended too. Looking through my code and by using W3Schools, I quickly realised I needed to switch all of my font sizes to EM and not PX, this helps scale them correctly across musltiple devices and browsers.
  • Furthermore, Another challenge I faced was when I ran into another scaling problem when using my webpage on mobile. Due to one of my elements using a Grid display when it was not nececessary, my website would load zoomed in and the only way to remove the horizontal scrolling along with viewing the webpage so that it fit within the devices viewport was by manually zooming the page out. This simple error took DAYS to figure out, after hours of looking through posts on Stack Overflow and reading my code back to front, I decided the only way I was going to identify the problem was to manually change each elements styling one by one. I knew this was going to take a while as my CSS had over 400 lines of code but I knew this technique would identify the root cause. Eventually after an hour or so of nitpicking my code, I found out the cuase of the bug was that the weekly posts class was styled 'grid' and not 'inline' causing the webpage to overflow out of the viewport and not scale correctly. This simple change fixed EVERYTHING and my website now was fully responsive on a number of different devices.
    I used the chrome developer tools in order to see how responsive my webage was across different screen sizes.
    I also viewed my webpage on different browsers, including Safari, Chrome and Firefox so that I knew how it appeared for users that may not be viewing the webpage how I may have been during development.
